About This Project

PID control is a concept that's straightforward on paper but notoriously tricky to develop an intuition for. This project set out to make that intuition tangible — a physically constrained quadcopter platform that lets you select different gain combinations and immediately see and measure the effect on system response. I was responsible for [your role here — e.g. the control system implementation and embedded software], working as part of a team to deliver a fully functional demonstrator.

Hardware
  • Modular quadcopter frame restricted to one degree of freedom (vertical motion only), fixed to four guide poles to prevent lateral or rotational movement
  • Four BLDC motors providing vertical thrust, driven by one ESC per two motors
  • LIDAR sensor providing altitude feedback to the controller
Control System
  • PID controller regulating vertical position, rising from a base position to a predefined reference height
  • User-selectable P, I, and D gain values, allowing multiple combinations to be tested without recompiling
  • Controller implemented in C++ running on the drone hardware
User Interface
  • Live graph of drone height against time, bridging the gap between theoretical step response curves and physical behaviour
  • Automatic calculation of rise time, overshoot, settling time, steady-state value, and max height for each flight
  • Graphing implemented in Python
