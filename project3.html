Machine Learning Control

Trimble AgIs Sponsored FYP

Overview

This project investigates the application of machine learning to vehicle steering control for autonomous agricultural guidance. Using the AgileX Hunter SE robotic platform integrated with Trimble's NAV-960 positioning system, the aim is to develop and evaluate a learning-based steering controller capable of robust path following under varying terrain and operating conditions.

Traditional control approaches in this domain rely on fixed models and high-precision sensing, but struggle to account for nonlinear vehicle dynamics caused by changing loads, terrain variability, and environmental conditions. The outcomes of this work contribute to ongoing research into next-generation autonomous guidance systems for agricultural machinery, with potential applications in precision farming to improve navigation accuracy, reduce soil disturbance, and increase operational efficiency.

Role & Responsibilities

This project is conducted in a team of five — three electrical engineering students and two mechatronics — with each member assigned a primary and secondary role to balance workload across the year. The five primary roles are simulation and plant modelling, controller design and development, machine learning development, systems integration, and testing and validation.

My primary role is systems integration, with a secondary focus on simulation and modelling. My technical responsibilities include:

  • Setting up the machine learning training and testing pipeline
  • Reading IMU and GNSS data from the NAV-960 module via CAN communication on a host computer
  • Sending and receiving control and telemetry data between the Hunter SE platform and onboard systems via CAN communication
  • Integrating the NVIDIA Jetson Xavier NX into the control pipeline as the onboard computing platform
  • Supporting the integration of simulation and physical testing environments
  • Validating simulation dynamics against the physical behaviour of the robotic platform
  • Assisting with controller deployment and real-time system testing

Project Stages

The project is currently at Stage 3 of the development plan.

Stage 1 — Simulation Model Development

Development of a Gazebo-based simulation of the AgileX Hunter SE platform to replicate vehicle dynamics and enable rapid testing of control strategies.

Stage 2 — Baseline Controller & Data Generation

Implementation of a baseline closed-loop controller in simulation for path following, used to generate state-action datasets for machine learning training.

Stage 3 — Imitation Learning Controller Development

Training of a behavioural cloning model to replicate baseline controller performance using inputs such as cross-track error, heading error, and look-ahead point data. Additional state variables and control history are progressively incorporated to improve performance.

Stage 4 — Reinforcement Learning Optimisation

Application of reinforcement learning techniques to refine the controller using reward functions that prioritise path accuracy, smooth steering, and stability, while penalising oscillatory behaviour.

Stage 5 — Disturbance Testing & Validation

Evaluation of controller robustness under disturbance conditions including uneven terrain and slopes. Performance is assessed using path-tracking accuracy, trajectory deviation, and control smoothness, with error distributions analysed using CDF metrics (p68 and p95).

Progress to Date

Simulation-to-ML Training Pipeline

Developed a data pipeline linking simulation outputs to the machine learning training workflow. This included implementing structured logging of state and control variables to CSV format, enabling automated dataset generation from simulation runs and integration into Python-based imitation learning training scripts. A workflow was also defined for exporting trained model parameters back into the simulation environment for iterative testing.

Simulation Validation & Steering Model Calibration

Assisted in validating the simulation model against the physical AgileX Hunter SE platform. This involved comparing simulated steering response with real-world vehicle behaviour and using experimental measurements to refine model parameters.

Onboard Compute & Communication Integration

Configured the Jetson Xavier NX development platform as the onboard computing environment, including OS setup, dependency configuration, and communication interface setup.

Robot Communication & CAN Data Pipeline

Implemented and tested CAN-based communication between the Hunter SE platform and onboard systems, including decoding and monitoring of real-time telemetry (steering angle, velocity, battery status, system state) at 500 kbaud. Ensured reliable data capture for both debugging and dataset generation.

GNSS/IMU Data Acquisition — NAV-960 Integration

Assisted in establishing a data pipeline for GNSS and IMU outputs from the Trimble NAV-960 system. Due to interface limitations, a temporary Arduino-based CAN-to-serial bridge was used to forward processed sensor data to the Jetson, enabling real-time monitoring and logging for system integration and validation.

The project is ongoing, with current work focused on implementing closed-loop control to complete the end-to-end system pipeline and enable validation of integrated sensing, communication, and control on the physical Hunter SE platform, alongside continued development of structured GNSS/IMU data handling for ML training support.
