PID Hovering Platform

ENEL300 Design Project

Overview

Developed for the third-year control systems course ENEL321, this hover platform was designed as a physical teaching tool to demonstrate the effect of PID gain tuning on real-world system response. The platform constrains a quadcopter frame to vertical motion only, allowing students to directly observe how different controller gain combinations affect transient and steady-state behaviour through live visualisation and measured response metrics.

The completed platform has been used as a live demonstration at multiple Electrical Engineering department open days, and integrated into ENEL321 lectures as a teaching aid for illustrating PID control concepts.

Role & Contributions

This project was completed in a team of three with clearly defined responsibilities. My primary contributions included software implementation, control system integration, PID gain tuning, and development of the real-time graphing and analysis interface. Hardware fabrication and electronics integration were handled by the other team members.

Project Overview

Hardware Setup
  • Modular quadcopter frame restricted to one degree of freedom (vertical motion only), fixed to four guide poles — one per arm — to prevent lateral or rotational movement
  • Platform starts at a base position and rises to a predefined reference height
  • Four BLDC motors providing vertical thrust, with one ESC per two motors
  • LIDAR sensor used to measure altitude for feedback
Control System
  • PID controller regulating vertical position
  • User-adjustable PID gains with predefined selectable values for Kp, Ki, and Kd

User Interface
  • Live graphing of drone height against time, bridging the gap between theoretical and practical
  • Adjustable gain levels for P, I, and D allowing for multiple combinations
  • Calculation of rise time, overshoot, steady-state value, settling time, and max height for each flight
  • Software written and implemented in C++ for drone control; graphing done in Python

Technical Highlights

These focus on the sections I implemented. For further detail on hardware and electronics please see the attached reports below.

Control
  • Three discrete gain levels (low, medium, high) for Kp, Ki, and Kd, designed to produce clearly distinguishable system behaviours for educational demonstration
  • Low settings enable isolated control modes (e.g. proportional-only response); medium settings provide a stable baseline PID configuration; high settings demonstrate extreme system responses such as overshoot and oscillation
User Interface
  • Real-time altitude response plotting during demonstrations, with recorded datasets saved for later replay and analysis
  • Potentiometer-based gain adjustment system allowing rapid configuration between flights
  • Automated calculation of key performance metrics including rise time, settling time, steady-state value, overshoot, peak value, and peak time
  • Steady-state estimation computed using a moving window average with tolerance-based convergence detection to improve metric accuracy
Software
  • LIDAR measurement noise characterised experimentally and filtered to improve control stability
  • Derivative term filtering implemented to reduce noise amplification effects
  • Sensor sampling rates evaluated experimentally; 40 Hz selected as the optimal reliable operating frequency
  • Modular software architecture separating control, sensing, and UI components

Results

The PID hover platform has been used in multiple Electrical Engineering department open day demonstrations, led by myself and teammate Brendan Booth. These sessions included a brief explanation of control systems, an overview of the project, and discussion of how PID control is applied in industry.

The system has also been used in the ENEL321 controls course (2026) to demonstrate PID gain concepts, led by Brendan Booth and myself alongside lecturer Chris Hann. This demonstration utilised the live graphing projected on screen for the students to see.

Feedback from this demonstration indicated that the hover platform improved student understanding of PID control, especially in relation to integral control behaviour. The platform has now been passed to lecturer Chris Hann to be used in future control courses.
