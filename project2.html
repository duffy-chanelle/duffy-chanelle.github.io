Antenna Tracking and Control

SpaceOps NZ Internship

Overview

Completed as part of a summer internship at SpaceOps NZ, this project focused on improving the pointing accuracy of the 30 m antenna at the Warkworth ground station. The existing field system and pointing model were experiencing pointing errors of up to 0.3°, which for a 30 m antenna can result in significant signal loss during spacecraft tracking operations. Previous mission-specific software had demonstrated that pointing errors as low as 0.002° were achievable, indicating that the antenna hardware itself was capable of substantially higher accuracy.

The objective of the project was to investigate the underlying causes of these pointing errors and support the development and validation of a higher-accuracy pointing system. Over the course of the internship, the project evolved to include both redevelopment of existing field-system functionality and development of new antenna tracking capabilities, including velocity-based spacecraft tracking control. The resulting software was subsequently used during tracking operations associated with the Artemis II launch campaign in April 2026.

Role & Responsibilities

The internship was conducted with a high degree of independence, with responsibility for managing investigation direction, development priorities, and technical progress. My work focused primarily on:

  • Investigating potential sources of pointing error within the existing field system, pointing model implementation, and control pipeline
  • Analysing field-system log data and validating coordinate transformation behaviour
  • Developing high-level Python software for antenna control, target tracking, scan automation, and structured data logging
  • Supporting validation and operational testing of the antenna control system
  • Developing and maintaining the project Concept of Operations (ConOps) documentation

The software development component was completed collaboratively with a computer engineer, who managed the low-level Modbus communication interface while I focused on control logic, coordinate transforms, and algorithm development.

Technical Highlights

Pointing Model Investigation
  • Investigated the implementation and behaviour of the existing antenna pointing model used within the field system
  • Analysed previous datasets and generated new pointing models for comparison against operational performance
  • Collected new datasets using the existing field system under varying environmental conditions
  • Investigated additional contributors to pointing error including timing offsets, coordinate transformation discrepancies, and data quality issues
  • Reverse engineered the current pointing-model generation software using produced logs, and wrote my own version for rapid generation of pointing models using pre-collected data
Software Development

Developed high-level Python software implementing key functionality of the existing field system:

  • Implemented velocity control logic for tracking and positioning the antenna
  • Developed coordinate handling functionality supporting both astronomical point-source observations (RA/Dec) and spacecraft ephemeris tracking
  • Implemented my own version of the field system's scan algorithm (Fivepoint), involving scanning across a source in both directions, taking measurements at each point, fitting a Gaussian curve to the measurements, and calculating the pointing error
  • Structured JSON-based data logging for pointing model training, validation, and debugging purposes
Validation & Testing
  • Conducted staged validation using both simulated antenna environments and physical antenna testing
  • Developed and tested safety mechanisms including hard/soft limits, velocity constraints, and weather-based shutdown
  • Performed structured testing using observational datasets and live satellite tracking to validate pointing behaviour and software reliability
Operational Deployment

A significant portion of the internship involved on-site testing at the Warkworth ground station. Development milestones were driven by active mission schedules, including CAPSTONE tracking for system validation as well as activities and preparation work associated with the Artemis II launch.

In addition to software development and validation, operational responsibilities included antenna stow procedures, site security, and supporting maintenance activities during live operations.
